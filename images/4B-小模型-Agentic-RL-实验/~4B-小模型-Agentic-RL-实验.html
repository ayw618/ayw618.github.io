4B 小模型 Agentic RL 实验

实验概述

实验目的探索小模型上的工具调用情况，并进行优化验证。以专家任务作为示例，构建相关的SFT和RL数据集，以及对应的工具。

  • 训练配置：4张A100
  • model: Qwen3-4B
  • SFT: XTuner
  • RL: VERL(GRPO)

多阶段实验设计

  • 第一阶段：生成无<tool_call>标签的SFT数据（10000条）用于冷启动
  • 第二阶段：生成有<tool_call>标签的SFT数据（10000条）用于冷启动
  • 第三阶段：生成含/不含<tool_call>标签的SFT混杂数据（5:5、6:4、8:3）用于冷启动

SFT逻辑

  • 构建数据集过程：构建10个不同的带工具的提示词模板，并在其中挖空，方便后续填充。比如，将一些数字挖空，后续使用相应的工具函数对提示词模板进行补充。

RL逻辑

  • 构建数据集过程：提示词模板与SFT相同；ground_truth部分使用工具进行解答，将答案填充进json数据。一共生成10000条RL数据。

  • RL数据格式：

RL多段奖励函数设计

采用RLVR，利用可验证奖励进行打分

第一部分（可选）

针对带<tool_call>标签的训练（实验的第二阶段与第三阶段）。如果没有输出<tool_call>，返回0分

第二部分（必选）

针对具体任务进行评分。根据不同任务，此部分可以设置不同奖励打分逻辑。

第三部分（可选）

针对冗余工具调用的打分逻辑。只有当前面的得分为1时，才进入该部分进行判断。逻辑如下：对传入的rollout进行字符串匹配，找到第一次出现正确答案的位置，根据在其之后出现的冗余<tool_call>调用进行扣分，冗余调用越多，扣分越多。

关键指标

  • num_turns/mean: 用来判断是否执行工具调用、是否进行冗余调用
  • critic/rewards/mean: 奖励得分。将同一批次的样本的所有rollout的得分取平均
  • critic/advantages/mean: 优势计算。可以判断训练是否收敛，当同一批次奖励得分全0或全1时，优势值为0
  • acc/mean@1: 评估得分

第一阶段

第一阶段SFT数据无 \ 标签。
奖励计算只包含第二部分

  • SFT 结果: loss 最终收敛在 0.022~0.025
  • RL结果: 准确率达到85%以上。推测是SFT数据太多了，在模型训练冷启动阶段，模型就已经学习到了大量的对应专家知识
  • num_turns = 2: 没有 user 的回复（即 tool response）。因此，num turns始终为2

  • acc_mean@1曲线: 可以发现初始准确率就达到了60%以上。后续收敛到85%~87%

  • critic/rewards/mean曲线: 训练到最后1250步时，rewards有下降趋势，而且从波动曲线来看，训练到1000~1100步时，便已经收敛

第二阶段

第二阶段SFT数据有 <tool_call> 标签。奖励计算与第一阶段相同
奖励计算只包含第二部分

  • SFT 结果: loss 最终收敛在 0.021~0.023
  • RL结果：准确率达到95%以上。耗时变长，因为进行了多次工具调用，生成的整个rollout长度变长，时间随之增加。
  • num_turns = 8: 有 user 的回复（即 tool response）。但是出现冗余调用，导致 num turns 始终为8

  • acc_mean@1曲线: 可以发现初始准确率就达到了85%以上。后续收敛到96%~98%

  • critic/rewards/mean曲线: 仅仅训练700步时，奖励得分、优势、交叉熵均达到收敛标准。相关评估得分也已经达到了95%以上


第三阶段

第三阶段SFT数据为混合数据。数据比例为：含<tool_call>标签: 不含<tool_call>标签 = 5: 5
奖励计算包含第一部分、第二部分、第三部分

  • SFT结果: loss 最终收敛在 0.028~0.030
  • RL结果: 准确率达到93%以上，最后收敛时，num turns为8
  • num_turns = 8: 有 user 的回复（即 tool response）。但是出现冗余调用，导致 num turns 始终为8。但是，本次实验中，添加了对 tool response 的奖励与冗余调用的惩罚，因此，num turns 在训练过程中会出现波动。
    • 从曲线可以看出：在950步之前，特别是900步之前，num turns都在进行正常的波动，在奖励的惩罚下，num turns 会尝试减少，但是会因为 tool response 的奖励而增加，导致 num turns 增加；在950步之后，num turns 收敛在8；

  • acc_mean@1曲线: 可以发现初始准确率就达到了65%以上。后续收敛到92%~94%

  • critic/rewards/mean曲线: 训练900步之后，奖励得分、优势、交叉熵均达到收敛标准。相关评估得分也已经达到了90%以上


后续本人还尝试了8:2的数据比例，但是在100步之后，num turns 始终为8。

结论

由于时间不足，未能进行更加详细的实验，目前只能进行一个粗糙的总结。
针对 4B 小模型的 Agentic RL 实验，以下是一些总结：

数据层面

根据实验结果，混合数据有一定效果，但是需要在数据比例上进行调整和测试。比如5:5、6:4、7:3、8:2的数据比例。

同时，根据经验，本人认为打乱数据也是必要的，以避免模型过拟合，以下是本人针对不打乱数据的情况进行分析。

  • 在GRPO算法中，如果5:5的数据不进行打乱，模型可能在前一半无<tool_call>标签数据训练时，学习到了不进行工具调用，进行后续的带<tool_call>标签的数据训练时，很有可能出现全0的情况（奖励采用第一部分），这会导致 advantage 为0，梯度为0，模型无法进行更新。
  • 如果采用DAPO算法呢？在当前的流程下，恐怕也没有用处，以下仅为个人推测，具体还需实验验证。
    • DAPO在GRPO上做了创新，提高clip ratio上限，鼓励模型探索新的策略。但这仅仅是“鼓励”，这对于模型初期训练可能有用，因为此时模型的“状态空间”相当于一片混沌（更加适合探索的情况是模型完全不进行SFT，直接进行RL训练），模型此时会不断尝试新的策略，以期找到能够得分的策略。
    • DAPO针对GRPO的梯度消失做了优化，引入了重试策略，让模型必须生成出不是全0或全1的情况。但是，在当前假定流程下，模型会自认为已经找到了可以得分的路径，让其探索的难度相较初始状态增加。
  • 如果采用OPD算法呢？这个可能比前两个算法更有用，因为有“外在力量”（教师模型）介入，不受学生模型影响。

训练层面

可以提前停止训练，比如在第三阶段实验中，可以只训练到800步就停止，让模型的num turns保持波动，不再收敛，同时，模型的准确率也保持在90%左右。

奖励设计层面

不断调整第一部分、第三部分的奖励设计参数，以期找到能够让其学会工具调用且拒绝冗余调用的策略。
